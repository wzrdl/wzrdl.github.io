## 4) 视觉与风格规范

* **字体**：`Inter`（正文/标签） + `Space Grotesk`（Hero 大字）；备选 `General Sans / Söhne` 风格。
* **字号层级**：

  * Hero 主标题：`clamp(48px, 8vw, 112px)` 粗体
  * Section 标题：`clamp(24px, 3vw, 40px)`
  * 正文：16–18px；元信息 13–14px
* **配色**（极简黑白 + 点缀橙/蓝 二选一）：

  * 背景：#FFFFFF / #0A0A0A（暗色）
  * 文字：#111111 / #EDEDED
  * 点缀：Orange `#FF5A1F`（靠近模板的小圆点）或 Electric Blue `#4C7DFF`
* **图像**：

  * 需要一张你的**黑白半身照**（高分辨率，主体居中，四周留白，PNG/JPG）；前景羽化到白
* **图标**：Lucide 或 Simple Icons（LinkedIn/GitHub/Email/外链箭头）
* **网格与留白**：Max-width 1200px，列间距 24–32px，区块垂直间距 96–128px
* **暗色模式**：系统偏好自动；切换按钮置于页脚

## 5) 交互与动画

* **滚动显隐**：区块进入视口时淡入/上移（AOS 或自写 IntersectionObserver + CSS transitions 200–400ms）
* **链接悬停**：轻微上移 + 阴影/下划线出现
* **CTA 按钮**：按压态缩放 0.98；focus 可见（3px 外发光）
* **顶栏**：保持粘顶，滚动后透明→白底阴影

## 6) 可访问性与 SEO

* 语义化结构（`<header> <main> <section> <article> <footer>`）
* 对比度 ≥ 4.5:1；键盘可达；`prefers-reduced-motion` 兼容
* `<title>`、`<meta name="description">`、Open Graph（og:image 使用你的半身照）
* 结构化数据（Person / WebSite）可选

## 7) 性能

* 纯静态，无重型框架；只用少量原生 JS
* 图片用 `srcset` 与 `loading="lazy"`；SVG 图标内联
* CSS 原子化（Tailwind）或手写少量 SCSS；生产压缩

## 8) 技术栈与项目结构（两套等价方案）

### A) 纯静态（最轻量，建议）

```

├─ index.html
├─ /assets
│  ├─ avatar.jpg (或 .png)     # 你的黑白半身照

├─ /styles
```

### B) Tailwind + 构建（更易控样式）

* 优点：快速迭代视觉；Tailwind JIT 与 `@layer` 自定义组件。

## 12) GitHub Pages 部署（两种方式）

### 方式一：用户主页站（`<username>.github.io`，强烈推荐）

1. 新建仓库名 **`wzrdl.github.io`**（仓库必须与用户名一致）。
2. 将上文项目结构的文件推送到主分支（`main`）。
3. 打开 **Settings → Pages**，Source 选择 `Deploy from a branch`（main / root）。
4. 数十秒后即可通过 `https://wzrdl.github.io/` 访问。
5. 若用自定义域名，在 **Pages → Custom domain** 里设置并添加 DNS `CNAME` 记录。

### 方式二：项目页（`/portfolio` 之类）

1. 任意仓库（如 `portfolio`），分支 `main`。
2. **Settings → Pages**，Source 选 `GitHub Actions` 或 `Deploy from a branch (docs/)`。
3. 访问 `https://wzrdl.github.io/portfolio/`。
